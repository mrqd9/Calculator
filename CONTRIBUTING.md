### Reporting Bugs
* Check the **Issues** tab to see if the bug has already been reported.
* If not, open a new issue. Include your device type, browser version, and steps to reproduce the bug.

### Suggesting Enhancements
* Open an issue titled "Feature Request: [Your Idea]".
* Describe why this feature would be useful for the "Pro" billing workflow.

### Pull Requests (Code Changes)
1. Fork the repository.
2. Create a branch for your fix (`git checkout -b fix/bug-name`).
3. **Important:** Ensure you do not remove the license headers from the top of `app.js`, `style.css`, or `index.html`.
4. Commit your changes and push to your fork.
5. Submit a Pull Request to the `main` branch.

## üé® Coding Standards
* Use clean, documented Vanilla JavaScript.
* Ensure any UI changes remain responsive and work in **Portrait mode** (landscape is restricted by the overlay).
* Keep the **Indian Number System** formatting intact for financial values.
